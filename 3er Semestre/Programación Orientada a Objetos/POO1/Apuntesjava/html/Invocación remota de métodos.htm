Invocación remota de métodos (RMI)

La interface RMI permite que objetos Java en diferentes hosts se comuniquen unos con otros. Cada objeto remoto implementa una interfaz remota que especifica cuales de sus métodos pueden ser invocados por los clientes. Los clientes pueden invocar los métodos del objeto remoto casi exactamente como invocan los métodos locales.

 

Desde el punto de vista del programador, los objetos remotos y métodos remotos trabajan como los objetos locales y métodos locales a los que esta acostumbrado. Solo tiene que importar un paquete, buscar el objeto remoto en un registro, y asegurarse de manejar las excepciones remotas (RemoteExceptions) cuando se llaman los métodos del objeto remoto. De ese punto en adelante, se puede usar el objeto remoto casi tan libre y fácilmente como se usa un objeto ejecutándose en el sistema local.

El modelo de capas de rmi

 

Para asegurar la compatibilidad con los programas y las implementaciones java existentes, y para hacer el proceso tan transparente como sea posible para el programador, la comunicación entre un cliente y un servidor de un objeto remoto es implementada como una serie de niveles como se muestra en la figura de arriba.

 

Ejemplos de programas RMI

Ejemplo 1

El lado del servidor

Para crear un objeto remoto, se debe definir primero una interfaz que extienda la interfaz java.rmi.Remote. Esta subclase de Remote determina cuales métodos del objeto remoto que se esta creando pueden ser llamados por clientes.

 

//la interface hello

import java.rmi.*;

 

public interface Hello extends Remote {

 

  public String sayHello() throws java.rmi.RemoteException;

 

}

 

Cada método en la sub-interfaz debe declarar que este lanza una excepción de la clase RemoteException.

 

Después de definir una interfaz remota, se debe definir una clase que implemente la interfaz remota.

 

import java.rmi.*;

import java.rmi.server.*;

import java.net.*;

 

public class HelloImpl extends UnicastRemoteObject implements Hello {

 

  public HelloImpl() throws RemoteException {

    super();

  }

   

  public String sayHello() throws RemoteException {

    return "Hola, Mundo!";

  }

 

  public static void main(String args[]) {

 

    try {

      HelloImpl h = new HelloImpl();

      Naming.rebind("hello", h);

      System.out.println("Servidor Hello listo.");

    }

     catch (RemoteException re) {

      System.out.println("Exception in HelloImpl.main: " + re);

    }

    catch (MalformedURLException e) {

      System.out.println("MalformedURLException en HelloImpl.main: " + e);

    }

 

  }

 

}

 

Para generar los stubs (del lado del cliente) y los esqueletos (lado del servidor) para el objeto remoto de la clase HelloImpl, ejecute rmic en el archivo  .class(punto class). Por ejemplo

 

%rmic HelloImpl

 

Con el comando ls (de unix) podemos ver los nombres de los archivos que rmic genero (HelloImpl_Skel.class y HelloImpl_Stub.class)

%ls Hel*

 

Hello.class       HelloImpl.class HelloImpl_Skel.class

Hello.java        HelloImpl.java  HelloImpl_Stub.class

 

Para que el servidor pueda registrarse con el registro de nombres, se debe iniciar dicho registro así:

 

%rmiregistry & (en unix)

o

c:>start rmiregistry (en windows)

Para ejecutar el servidor

%java HelloImpl &

Servidor hello Listo

 

Del lado del cliente

Antes de que un cliente pueda llamar a un método remoto, este necesita obtener una referencia remota al objeto remoto. Un programa obtiene una referencia remota solicitando un registro en el servidor para el objeto remoto. Dicho programa lo solicita llamando el método lookup(String url).

 

 

//listado del cliente

import java.rmi.*;

 

public class HelloClient {

 

  public static void main(String args[]) {

       

    System.setSecurityManager(new RMISecurityManager());

 

    try {

 

      Hello h = (Hello) Naming.lookup("hello");

 

      String message = h.sayHello();

      System.out.println("HelloClient: " + message);

    }

    catch (Exception e) {

      System.out.println("Exception in main: " + e);

    }

 

  }

 

}

 

el cliente se ejecuta con

%java HelloClient

HelloClient: Hola, Mundo!

 

A continuación se muestra (como ejemplo) el juego de gato en red implementado ahora usando rmi para esto usaremos dos tipos de objetos remotos, un tipo lleva el control de las jugadas (el objeto remoto que hace de servidor) y otro para los jugadores (estos son los objetos que hacen el papel de jugadores del gato y también el papel de clientes del servidor del gato).

 

Empezamos por la interfaz que extiende la interfaz java.rmi.Remote (la cual contiene los métodos que se pueden llamar de manera remota). Esta interfaz la usa el objeto que contola las jugadas.

 

import java.rmi.*;

 

public interface GatoServer extends Remote {

      public void addPlayer (GamePlayer gp) throws RemoteException;

      public void  broadCast(GamePlayer gp, int pos) throws RemoteException;

}

 

 

Después seguimos con la clase que implementa dicha interfaz remota

 

import java.rmi.server.*;

import java.rmi.*;

import java.util.*;

 

public class GatoServerImpl extends UnicastRemoteObject implements GatoServer {

      private Vector players=new Vector();

 

      public GatoServerImpl() throws RemoteException {}

      public static void main(String [] args){

            try {

                  GatoServerImpl gatoS=new GatoServerImpl();

                  Naming.rebind("GatoServer", gatoS);

            }

            catch(Exception ex){

            }

      }

      public void addPlayer(GamePlayer gp) throws RemoteException {

            players.addElement(gp);

      }

      public synchronized void broadCast(GamePlayer gp, int pos)throws RemoteException {

            for(int i=0; i< players.size(); i++){

                  GamePlayer g=(GamePlayer)(players.elementAt(i));

                  if(!g.equals(gp)){

                        g.mueve(pos);

                  }

            }

      }

}

 

Luego viene la interfaz que extiende la interfaz java.rmi.Remote para el jugador

 

import java.rmi.*;

 

public interface GamePlayer extends Remote {

      public void mueve(int pos) throws RemoteException;

}

 

 

Y después la clase que implementa la interfaz anterior

 

import java.rmi.server.*;

import java.rmi.*;

 

public class GamePlayerImpl extends UnicastRemoteObject implements GamePlayer {

      private GameClient gameClient=null;

      public GamePlayerImpl(GameClient gc) throws RemoteException {

            this.gameClient=gc;

      }

      public void mueve(int pos)throws RemoteException {

            gameClient.mueve(pos);

      }

}

 

 

Y finalmente una clase auxiliar que se ocupa de la interfaz gráfica y del crear el objeto remoto que hace el papel del jugador.

 

import java.awt.*;

import java.awt.event.*;

import java.applet.*;

import java.net.*;

import java.rmi.*;

import java.util.*;

public class GameClient extends Applet implements ActionListener{

      private GamePlayer gamePlayer;

      private GatoServer gatoServer;

      private Button botones[];

      private boolean turno=true;

      private String hostname="localhost";

      public void mueve(int pos){

            String s="";

            if(turno) s="X"; else s="O";

            botones[pos].setLabel(s);

            botones[pos].setEnabled(false);

            turno=!turno;

      }

      public void init(){

            setLayout(new GridLayout(3,3));

            botones=new Button[9];

            for(int i=0;i<9;i++){

                  add(botones[i]=new Button(""+i));

                  botones[i].addActionListener(this);

            }

            try {

                  gamePlayer=new GamePlayerImpl(this);

                  connect();

            }

            catch(Exception ex){ System.err.println(ex); }

      }

     

 

 

 

 

protected void connect() throws RemoteException,

      MalformedURLException, NotBoundException {

            gatoServer=(GatoServer)Naming.lookup

            ("//"+hostname+"/GatoServer");

            gatoServer.addPlayer(gamePlayer);

      }

      public void actionPerformed(ActionEvent e){

            String s="";

            Button b=(Button)e.getSource();

            if(turno) s="X"; else s="O";

            b.setLabel(s);

            b.setEnabled(false);

            turno=!turno;

            for(int i=0; i<9;i++)

                  if(botones[i]==b)

                    try {

                        gatoServer.broadCast(gamePlayer, i);          

                    }

                    catch(RemoteException re){

                        System.err.println(re);

                    }

      }

      public static void main(String args[]){

            GameClient g=new GameClient();

            g.init();

            g.start();

            Frame f=new Frame("Gato");

            f.add("Center", g);

            f.pack();f.show();

      }
